STUDENTORACLEAPP.JAVA – EXPLANATION

1) Overview
StudentOracleApp is a Java Swing (GUI) application that connects to an Oracle Database and performs basic CRUD operations (Create, Read, Update, Delete) on a STUDENT table.
It also calculates the average of three marks for a given student ID and displays all students in a JTable.

Main features:
- Connects to Oracle using JDBC (Oracle Thin driver)
- Displays all students in a table (JTable)
- Calculates average of mark1, mark2, mark3 for a given student ID
- Menu window to Insert / Delete / Update students
- Prints DB actions and student details to the console for verification


2) Database Assumptions
The program expects a table named STUDENT with these columns (order matters because the code uses SELECT * and positional indexes):
- stdid  (NUMBER / INT)  PRIMARY KEY
- name   (VARCHAR2)
- mark1  (NUMBER / INT)
- mark2  (NUMBER / INT)
- mark3  (NUMBER / INT)

Example table definition (reference):
CREATE TABLE student (
  stdid NUMBER PRIMARY KEY,
  name  VARCHAR2(50),
  mark1 NUMBER,
  mark2 NUMBER,
  mark3 NUMBER
);


3) GUI Layout (Main Window)
The main JFrame contains:
- "Enter Student ID" text field (idField)
- "Calculate Avg" button (avgBtn)
- output field for average (avgField, read-only)
- "Menu" button (menuBtn)
- JTable to show all students from DB

Startup behavior:
- connectDB() is called
- if connection succeeds -> loadTable() fills JTable with all rows
- if connection fails -> buttons are disabled and an error dialog is shown


4) JDBC Connection (connectDB)
Method: connectDB()
- Loads Oracle JDBC driver: Class.forName("oracle.jdbc.OracleDriver")
- Connects using:
  url  = jdbc:oracle:thin:@localhost:1521:xe
  user = system
  pass = 1229
- If it fails, con is set to null and a JOptionPane error is displayed.

Note:
- Credentials are hard-coded (for lab/demo). In real projects, keep them in config and never hard-code passwords.


5) Table Loading (loadTable)
Method: loadTable()
- Clears the existing table model rows: model.setRowCount(0)
- Executes: SELECT * FROM student
- For each row, adds to DefaultTableModel as: [id, name, m1, m2, m3]
- This method is called after every Insert/Delete/Update to refresh the GUI.


6) Average Calculation (calculateAvg)
Method: calculateAvg()
Flow:
- Reads student ID from idField
- Executes a prepared query:
  SELECT mark1, mark2, mark3 FROM student WHERE stdid=?
- If found, computes integer average:
  avg = (m1 + m2 + m3) / 3
  and displays it in avgField.
- If not found, shows "Student not found!" and clears avgField.
- If input is invalid (empty/non-number), shows "Invalid Input!".


7) Menu Window (MenuWindow)
Inner class: MenuWindow extends JFrame
- Opens a small window with 3 buttons:
  Insert -> opens InsertForm
  Delete -> calls deleteStudent()
  Update -> opens UpdateForm


8) Insert Operation (InsertForm)
Inner class: InsertForm extends JFrame
- Collects: ID, Name, Mark1, Mark2, Mark3
- On Save:
  - Prepares and executes:
    INSERT INTO student VALUES(?,?,?,?,?)
  - Prints the inserted student details to console
  - Calls loadTable() to refresh JTable
  - Calls printAllStudents() to display all records in console
  - Closes the insert window (dispose())


9) Delete Operation (deleteStudent)
Method: deleteStudent()
- Prompts user for ID using JOptionPane input dialog
- Confirms deletion using JOptionPane confirm dialog
- Before deleting, prints the student details (if found) with message "will be deleted..."
- Executes:
  DELETE FROM student WHERE stdid=?
- If rows > 0 -> prints "deleted from DB." and prints all students in console
- If rows == 0 -> shows "Student not found!"
- Calls loadTable() to refresh JTable


10) Update Operation (UpdateForm)
Inner class: UpdateForm extends JFrame
- Inputs:
  - ID to update
  - New Name
  - New Mark1, Mark2, Mark3
- Executes:
  UPDATE student SET name=?, mark1=?, mark2=?, mark3=? WHERE stdid=?
- If rows > 0 -> prints updated student details and prints all students
- Else -> shows "Student not found!"
- Calls loadTable() to refresh JTable and closes window


11) Console Print Helpers
These methods are used only for printing to the console:
- printStudentDetails(...): prints one student record + action line
- printStudentById(id, actionLineIfFound): fetches a student by id and prints
- printAllStudents(): prints all records in sorted order

Purpose:
- Helpful for lab output verification alongside the GUI table.


12) How to Run (Lab Setup)
1) Ensure Oracle XE is running and listener is on port 1521
2) Ensure the STUDENT table exists in the connected schema
3) Add Oracle JDBC driver (ojdbc) to the classpath
4) Compile and run StudentOracleApp

If you see "Connected to Oracle DB" in console, connection is successful.


13) Important Notes / Limitations
- Uses absolute layout (setLayout(null) + setBounds). Works for lab, but resizing/spacing is not responsive.
- Average uses integer division (no decimal). Example: (80+81+82)/3 = 81 (integer).
- Database credentials are hard-coded.
- Connection is not explicitly closed when app exits (acceptable for short lab runs). In production, you would close it on window close.
